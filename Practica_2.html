Practica 2 Diagramas de Clases UML

Modelado de clases, relaciones y decisiones de diseno para representar soluciones orientadas a objetos.

1) Mapa Mental

Desarrolle un mapa mental comparando programacion estructurada y POO. Incluya modularidad, reutilizacion, mantenimiento, pruebas y escalabilidad.

2) Elabore diagramas UML (atributos, metodos, visibilidad y relaciones). En cada caso agregue una nota con supuestos de diseno.

A) Circunferencia

  • Atributo radio.
  • Metodos para area y perimetro.
  • Propiedades para modificar y consultar radio.

B) Rectangulo

  • Datos: Largo, Ancho.
  • Metodos: CalcularArea(), CalcularPerimetro().

C) Conversor Temp

Disene clase para convertir temperatura F a C.

D) Tipos de Objetos

  1. Navegador de internet.
  2. Factura de ventas.
  3. Computadora.

Para cada uno: minimo 3 atributos y 3 acciones.

E) Taller Mecanico (Modelado Complejo)

Modelar orden de trabajo con id unico, datos de cliente (nombre, direccion, telefono) y datos del auto (marca, modelo, color, placas), mostrando relaciones entre clases.

F) Clase Fecha

  • Datos: Dia, Mes, Ano.
  • Constructores: por sistema o parametros.
  • Metodos: modificar, mostrar formato y validar fecha.

G) Supermercado

  • Clave, Descripcion, Precio, Stock.
  • Constructor con parametros.
  • Metodo para modificar 4 atributos y propiedades individuales.

H) Cuenta Bancaria

  • Atributo: Monto.
  • Operaciones: Depositar, Girar, Obtener Saldo.

I) Aerolinea

  • Numero, Fecha, Hora despegue, Duracion.
  • Operaciones: demora y fecha/hora de aterrizaje.

Rubrica Sugerida para UML

Correccion Tecnica (35%)

Atributos, metodos, visibilidad y firmas coherentes.

Modelado (30%)

Relaciones correctas: asociacion, composicion, agregacion.

Claridad (20%)

Diagrama legible y nombres significativos.

Justificacion (15%)

Explicacion breve de decisiones de diseno.
